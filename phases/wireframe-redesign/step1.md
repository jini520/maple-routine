# Step 1: bottom-tab-bar

## 읽어야 할 파일

먼저 아래 파일들을 읽고 프로젝트의 아키텍처와 설계 의도를 파악하라:

- `src/App.tsx` — 현재 상단에 텍스트 링크(`일간`/`주간`) 두 개짜리 nav가 있다. 이번 step이 이걸 교체한다. `AppShell`이 `restoreFromStorage()`를 마운트 시 1회 호출하고 `status`에 따라 `/`, `/onboarding`, `/daily`, `/weekly` 라우트를 가드하는 구조를 그대로 유지해야 한다 — 라우팅 로직 자체는 건드리지 말고 내비게이션 UI만 바꿔라.
- `/docs/UI_GUIDE.md` — 색상 토큰

## 배경

사용자가 제공한 실제 와이어프레임을 검토한 결과, 내비게이션은 화면 하단에 고정된 **탭바**였다(상단 텍스트 링크가 아니다). 탭바 시각 패턴(그대로 옮김):
- 화면 하단에 고정, 위쪽에 얇은 보더 구분선(`border-t border-[#F0DFD1]`)
- 탭 항목들은 가로로 균등 배치(`justify-around` 또는 `justify-evenly`)
- 각 탭 항목은 세로로 아이콘(위) + 라벨(아래) 배치
- 아이콘은 아직 실제 아이콘 시스템이 정해지지 않아(UI_GUIDE.md 아이콘 섹션 미작성) 와이어프레임에서도 작은 사각형 테두리 플레이스홀더로만 표시했다 — 이번 step도 아이콘 라이브러리를 새로 추가하지 말고 같은 방식(작은 사각형 테두리 박스)으로 자리만 잡아둬라.
- 비활성 탭: 흐린 텍스트색(`text-[#B7A490]`). 활성 탭(현재 라우트와 일치): 강조 텍스트색(`text-[#C2410C]`)

**와이어프레임은 5개 탭(일간/주간/타이머/수익/드랍)을 그렸지만, 지금 실제로 존재하는 화면은 일간·주간뿐이다.** 타이머·수익·드랍은 아직 `features/`도 `app/`도 없다 — 존재하지 않는 화면으로 가는 죽은 탭을 만들지 말고, **일간·주간 2개 탭만** 렌더링하라. 나머지 3개는 그 기능들의 화면을 만드는 후속 task에서 같은 탭바에 추가하면 된다. 마찬가지로 와이어프레임의 우상단 톱니바퀴(설정) 아이콘도 설정 화면이 아직 없으므로 이번 step에서 만들지 마라.

## 작업

`src/App.tsx`의 `AppShell`을 수정하라:
- 기존 상단 텍스트 링크 nav를 제거한다.
- `status === 'completed'`일 때만 화면 **하단**에 고정된 탭바를 렌더링한다(온보딩 화면에는 탭바가 없다).
- 탭바는 일간(`/daily`)·주간(`/weekly`) 2개 항목만 가진다. 각 항목은 아이콘 자리(작은 사각형 테두리 박스) + 라벨("일간"/"주간")을 세로로 배치하고, `react-router-dom`의 `useLocation`(또는 동등한 방법)으로 현재 경로와 일치하는 탭에 활성 스타일을 적용한다.
- 탭바가 화면 콘텐츠를 가리지 않도록, 라우트로 렌더링되는 메인 콘텐츠 영역에 탭바 높이만큼 하단 여백(padding-bottom)을 줘라.

## 검증 절차

1. 위 AC 커맨드를 실행한다.
2. 아키텍처 체크리스트를 확인한다:
   - 라우트 가드 로직(`/`, `/onboarding`, `/daily`, `/weekly`의 `status` 기반 리다이렉트)을 그대로 유지했는가?
   - 존재하지 않는 화면(타이머/수익/드랍)으로 가는 탭이나 설정 아이콘을 만들지 않았는가?
3. `src/__tests__/App.test.tsx`를 이번 변경에 맞게 갱신하라(TDD — 테스트를 먼저 고친 뒤 구현을 맞춰라). 최소한 다음을 검증하라:
   - `status: 'completed'`이고 현재 경로가 `/daily`이면 탭바에서 "일간" 탭이 활성 스타일이다.
   - `status: 'completed'`이고 현재 경로가 `/weekly`이면 "주간" 탭이 활성 스타일이다.
   - `status !== 'completed'`(예: `'awaitingApiKey'`)이면 탭바 자체가 렌더링되지 않는다.
   - 탭바에 "타이머"/"수익"/"드랍" 텍스트가 없다.
4. 결과에 따라 `phases/wireframe-redesign/index.json`의 step 1을 업데이트한다:
   - 성공 → `"status": "completed"`, `"summary"`에 변경 내용을 한 줄로 요약
   - 3회 시도 후에도 실패 → `"status": "error"`, `"error_message"`에 구체적 에러
   - 사용자 개입 필요 → `"status": "blocked"`, `"blocked_reason"` 기록 후 즉시 중단

## 금지사항

- 타이머·수익·드랍 탭이나 설정 아이콘을 만들지 마라. 이유: 해당 화면들이 아직 존재하지 않아 죽은 링크가 된다 — 그 기능들의 후속 task에서 추가한다.
- 아이콘 라이브러리(예: lucide-react, react-icons 등)를 새로 설치하지 마라. 이유: 아이콘 시스템은 아직 미정(`UI_GUIDE.md` 아이콘 섹션 미작성)이라 와이어프레임과 같은 플레이스홀더로 자리만 잡아둔다.
- `/`, `/onboarding`, `/daily`, `/weekly` 라우트 가드 로직 자체를 바꾸지 마라. 이유: 이번 step은 내비게이션 UI 교체만 다룬다.
- 기존 테스트를 깨뜨리지 마라(단, 상단 nav 관련 기존 테스트는 이번 변경에 맞게 고쳐도 된다).
