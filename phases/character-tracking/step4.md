# Step 4: boss-profit-policy-doc

## 읽어야 할 파일

먼저 아래 파일들을 읽고 프로젝트의 아키텍처와 설계 의도를 파악하라:

- `/docs/PRD.md`의 "4. 주간 보스 수익 계산기" 섹션(약 49~58번째 줄) — "처치한 보스 목록은 주간 스케줄러와 동일하게 Nexon API 동기화 데이터를 그대로 사용"이라는 문장이 이미 있다.
- `/docs/ARCHITECTURE.md`의 "데이터 흐름" 섹션 중 "features/boss-profit, features/item-drop도 같은 캐시에서 '등록된 주간 보스 목록'만 읽기 전용으로 구독한다"는 문장(69번째 줄 근처)과 "보스 수익 계산기는 결정 판매 수익 + 물욕템 환산 가치... 표시"라는 문장(89번째 줄 근처).

## 배경

**이번 step은 코드를 전혀 바꾸지 않는다 — 문서만 정정한다.** `boss-profit`(주간 보스 수익 계산기) 기능 자체는 아직 구현되지 않았다(이 task에 포함되지 않음, 나중에 별도 task로 만든다).

사용자가 확정한 정책: 보스 수익 계산기에는 **실제로 처치한 보스만 표시**하고, 수익도 **처치 기준**으로 산정한다. 지금 `ARCHITECTURE.md` 69번째 줄 근처는 `boss-profit`과 `item-drop` 둘 다 "등록된(registration_flag) 보스 목록"을 구독한다고 뭉뚱그려 서술돼 있는데, 이는 `item-drop`(물욕 아이템 — 등록만 해도 드랍 기록 대상이 될 수 있다)에는 맞지만 `boss-profit`(수익 계산은 이미 죽인 보스에 대해서만 의미가 있다)에는 맞지 않는다. `PRD.md`는 이미 "처치한 보스 목록"이라고 정확히 서술돼 있으니, `ARCHITECTURE.md`를 그 표현에 맞춰 정정한다.

## 작업

`docs/ARCHITECTURE.md`의 "데이터 흐름" 섹션에서 "features/boss-profit, features/item-drop도 같은 캐시에서 '등록된 주간 보스 목록'만 읽기 전용으로 구독한다([[ADR-011]]) — 이 목록 자체는 편집 불가, 사용자가 남기는 기록(파티원 수·아이템 획득)만 별도로 storage/에 쓴다" 문장을, 기존 문서 스타일(`~~취소선~~` + 날짜 정정)을 따라 다음 취지로 수정하라:
- `item-drop`은 기존 그대로 "등록된(`registration_flag: true`) 주간 보스 목록"을 구독한다.
- `boss-profit`은 그와 달리 "처치된(`complete_flag: true`인) 보스만" 구독·표시하고, 수익 계산도 처치된 보스만을 대상으로 한다(등록만 하고 아직 안 잡은 보스는 수익 계산기에 나타나지 않는다) — 사용자가 2026-07-11 확정한 정책.
- 이 목록 자체는 (기존과 동일하게) 편집 불가하고, 사용자가 남기는 기록(파티원 수 등)만 `storage/`에 쓴다는 서술은 유지하라.

정확한 문장 위치·표현은 재량이되, "등록된"과 "처치된"을 기능별로 명확히 구분하는 게 핵심이다. `PRD.md`의 해당 서술("처치한 보스 목록...")은 이미 맞는 내용이니 손대지 않아도 된다.
