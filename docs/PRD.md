# PRD: 메이플 루틴 (Maple Routine)

> 이 문서는 초안입니다. 하이브리드 프레임워크(Capacitor), 웹 프레임워크(Vite+React), 백그라운드 상시 알림 요구사항은 확정되었습니다. 기록 데이터는 로컬 저장 전용이나 일간/주간 진행 상태는 Nexon Open API 연동으로 조회합니다([[ADR-007]], 2026-07-09). 최상위 화면은 "일간/주간 스케줄러"에서 "컨텐츠/보스 스케줄러"로 개편되었습니다(각 화면 내부에 일간·주간·월간 탭, [[ADR-013]], 2026-07-11). 별도 공개 웹 서비스 제공은 보류, 게임 레퍼런스 데이터는 `src/data/`에 반영 완료. 보스 스케줄러에 캐릭터+보스+난이도 단위 상시 파티 인원 설정("파티 관리")이 추가되어 보스 카드의 솔로/파티 표시·필터링과 보스 수익 계산기의 파티원 수 기본값에 함께 쓰인다([[ADR-019]], 2026-07-13, 설계만·구현 전).

## 목표
메이플스토리 플레이어가 일간/주간 콘텐츠 소진, 사냥 중 스킬 재사용 타이밍, 주간 보스 수익, 물욕템 드랍 기록을 캐릭터/계정 단위로 한 곳에서 관리하도록 돕는다.

## 사용자
- 메이플스토리를 플레이하며 일간·주간 콘텐츠를 여러 캐릭터(본캐+부캐)에 걸쳐 반복 수행해야 하는 유저
- 주간 보스 파티 수익을 캐릭터별/계정별로 파악하고 싶은 유저
- 장시간 사냥 중 솔 야누스 등 주기적 스킬 재사용을 놓치지 않아야 하는 유저

## 플랫폼
웹앱으로 개발 후 Android/iOS에 WebView 하이브리드 앱으로 패키징. 하이브리드 프레임워크는 Capacitor로 확정 ([[ADR-001]] 참고). 백그라운드 알림/상시 타이머 표시가 핵심 기능이므로 순수 웹(브라우저 PWA)만으로는 요구사항을 충족하지 못한다고 보고 네이티브 래핑을 전제로 함.

**전제 조건 (2026-07-09 추가, [[ADR-007]])**: 일간/주간 스케줄러 기능을 쓰려면 사용자가 (1) 게임 클라이언트 자체의 "스케줄러" 기능에 추적하고 싶은 퀘스트·보스를 먼저 등록해두어야 하고, (2) 앱 설정 화면에 `openapi.nexon.com`에서 본인이 직접 발급받은 개인 API 키를 입력해야 한다. NEXON Open ID(OAuth 로그인)는 "메이플스토리 파트너스" 승인이 필요한 제한된 기능이라 사용하지 않는다 — 앱 자체의 회원가입/계정 시스템도 두지 않는다.

**온보딩 (확정, 2026-07-09)**: API 키를 입력하면 `character/list`로 캐릭터 목록을 조회한다. 하나의 Nexon 계정에 여러 "메이플 ID"가 있을 수 있어 응답에 계정이 2개 이상 나올 수 있는데, 이 경우 **어느 메이플 ID를 쓸지 선택하는 화면**을 보여준다. 선택은 이후 설정 화면에서 언제든 바꿀 수 있다. API 키 발급 안내는 앱 안에서 단계별 위저드를 만들지 않고, openapi.nexon.com 링크 + 간단한 샘플 이미지 + 설명 문구로 가볍게 안내한다. **확정(2026-07-12, [[ADR-016]])**: 계정이 확정되면 온보딩을 바로 끝내지 않고, 그 계정의 전체 캐릭터 데이터(이미지·공개 여부·일정)를 진행률 바와 함께 미리 받아온다("예열") — 이후 컨텐츠/보스 스케줄러·캐릭터 관리 피커에 처음 들어가도 로딩 없이 바로 뜨게 하기 위함. 캐릭터가 많은 계정은 이 과정이 몇 초 이상 걸릴 수 있음을 사용자가 받아들임(확정).

## 핵심 기능

### 1. 컨텐츠 스케줄러 ([[ADR-013]]로 화면 구조 개편, 2026-07-11 — 기존 "일간 스케줄러"·"주간 스케줄러"의 콘텐츠(퀘스트) 부분을 한 화면으로 통합)
- 화면 안에 **일간 탭**(기존 `daily_contents`)과 **주간 탭**(기존 `weekly_contents`)만 있다. **월간 탭은 없음** — Nexon 스케줄러 API에 월간 주기 일반 콘텐츠 자체가 없음
- **확정(2026-07-11, [[ADR-012]]·[[ADR-013]])**: 계정의 전체 캐릭터가 아니라 사용자가 화면 안 "캐릭터 관리"에서 직접 고른 캐릭터만 표시한다(저장해야 확정). 아직 고르지 않았으면 "캐릭터를 선택해주세요" 빈 상태로 표시. 이 추적 목록(`trackedCharacters:content`)은 보스 스케줄러와 독립적이며, Nexon 스케줄 API 호출 자체도 추적 대상 캐릭터로만 제한한다(계정 전체를 순차 동기화하지 않음 — 캐릭터가 많은 계정에서 로딩 속도 개선). **정정(2026-07-12, [[ADR-015]])**: ~~체크박스로 선택~~ → "캐릭터 관리" 피커의 후보 목록은 레벨 내림차순 정렬 + 캐릭터 이미지 카드형 그리드로 표시하고 카드 클릭으로 선택을 토글한다. 저장이 있어야 확정되는 방식은 동일. 이미지는 `character/basic` API를 후보 전체 ocid에 대해 병렬 호출해 가져오고, 정보 비공개(`access_flag: false`) 캐릭터는 후보 목록에서 제외한다.
- 캐릭터별로 일간/주간 콘텐츠 진행 상태를 Nexon Open API(스케줄러 Open API)로 동기화해 읽기 전용으로 표시. 앱 안에서 수동 체크는 하지 않음(확정, 2026-07-09)
- 표시되는 항목은 사용자가 게임 클라이언트의 "스케줄러" 기능에 실제로 등록해둔 것만 해당(`registration_flag: "true"`). 앱 자체에서 항목을 추가/등록할 수 없음
- 진행률은 API의 `now_count`/`max_count`를 그대로 표시(예: 몬스터파크 7/14)
- 지정 시각까지 미완료 항목이 있으면 로컬 알림 발송. 판단 기준은 마지막 캐시가 아니라 **알림 시각에 Nexon API를 실시간 재호출**해 정확히 확인(확정, 2026-07-09 — [[ADR-004]]). 배터리/네트워크 사용이 늘고, iOS는 정확히 그 시각에 재확인이 안 될 수 있음(베스트 에포트)
- 알림 발송 시각은 사용자가 설정 가능
- **확정(2026-07-09)**: "사용자 커스텀 항목 추가"는 완전히 제거. 게임에 등록되지 않은 항목은 API로 완료 여부를 추적할 수 없어 이 모델과 맞지 않음 — 표시되는 항목은 항상 게임 내 스케줄러 등록 목록과 정확히 일치
- **엣지 케이스 — 게임 내 스케줄러 미등록**: 신규 캐릭터 또는 스케줄러를 아직 써본 적 없는 캐릭터는 모든 항목이 `registration_flag: "false"`로 옴. 이건 에러가 아니라 정상 상태이므로, "표시할 항목이 없습니다 — 게임에서 먼저 스케줄러에 등록해주세요" 같은 빈 상태 화면으로 처리 (일간·주간 탭 공통, [[ADR-008]])
- **에러 상태**: 동기화 실패 시 마지막으로 성공한 데이터를 유지하고 "마지막 동기화 n분 전" 표시 + 새로고침 버튼. API 키가 무효화된 경우 설정 화면 재입력 유도 배너 ([[ADR-008]])
- **UI 통일(확정, 2026-07-13, [[ADR-018]])**: 일간/주간 탭의 활성 상태 표시는 보스 스케줄러와 동일한 pill 스타일(`rounded-full bg-primary/15 text-primary`)을 재사용한다. 드롭다운과 탭은 별도 카드로 묶지 않는다.

### 2. 보스 스케줄러 ([[ADR-013]]로 화면 구조 개편, 2026-07-11 — 기존 "주간 스케줄러"의 보스 섹션을 독립 화면으로 분리)
- 화면 안에 **주간 탭**(`cycle: "bossWeekly"`)과 **월간 탭**(`cycle: "bossMonthly"`, 현재 검은마법사 1종)이 있다. **일간 탭은 없음** — `bossDaily`(힐라 하드·핑크빈 카오스 등 일간으로 격하된 보스)는 [[ADR-007]] 정책대로 계속 완전히 무시([[ADR-013]]에서 재확인, 정책 재검토 아님)
- **확정(2026-07-11, [[ADR-012]]·[[ADR-013]])**: 컨텐츠 스케줄러와 동일하게 "캐릭터 관리"로 고른 캐릭터만 표시하고 Nexon 스케줄 API 호출도 그 캐릭터로만 제한한다. 단, 이 추적 목록(`trackedCharacters:boss`)은 컨텐츠 스케줄러와 **독립적**이다(예: 컨텐츠에서 안 고른 캐릭터를 보스에서 고를 수 있음). "캐릭터 관리" 피커 UI(레벨순 정렬·이미지 카드형)는 컨텐츠 스케줄러와 동일한 컴포넌트를 공유한다([[ADR-015]])
- 캐릭터별로 보스 진행 상태를 Nexon Open API로 동기화해 읽기 전용으로 표시(컨텐츠 스케줄러와 동일한 모델, 엣지 케이스·에러 상태도 동일)
- 기존에 검토했던 "앱 내에서 최대 12마리 선택" UI는 두지 않음. 주간 탭은 `cycle: "bossWeekly"`, 월간 탭은 `cycle: "bossMonthly"`인 항목만 각각 표시하고 완료 여부(`complete_flag`)를 그대로 표시. `src/data/weekly-bosses.json`은 API 응답의 보스명·난이도 표기(영문/공백 차이, "시즌 보스 메이린" 같은 별칭 포함)를 우리 데이터와 매핑하는 참조 테이블로만 쓰임(주간·월간 보스 모두 이 파일의 `weekly`/`eventWeekly`/`monthly` 섹션에서 조회). 매핑 안 되는 보스명은 원문 그대로 "알 수 없는 콘텐츠"로 표시(크래시 금지, [[ADR-008]])
- **게임 규칙 — 주간 보스 12마리 제한 (확정, 2026-07-09)**: 캐릭터당 주간 보스는 최대 12마리(난이도 조합 단위)까지만 처치 가능한 게 게임 자체 규칙이며, API의 `weekly_boss_clear_count`/`weekly_boss_clear_limit_count`가 이 카운트를 그대로 반영한다. **이 카운트는 주간 탭에서만 표시한다** — 월간 보스(검은마법사)는 이 12마리 제한과 무관하다([[ADR-013]]). "시즌보스"(현재는 메이린만 해당, 카이는 시즌 종료로 이미 제외)는 이 제한에서 예외라 진행률 표시(예: "n/12")에서 별도로 다뤄야 함
- 지정 시각까지 미완료 항목이 있으면 로컬 알림 발송(컨텐츠 스케줄러와 동일하게 알림 시각 실시간 재확인 기준, 확정)
- ~~엣지 케이스 — 리셋 경계~~ — 확인 완료(2026-07-09): Nexon 서버는 KST 목요일 00:00(주간)에 초기화되어 로컬 계산 리셋 시각과 정확히 일치. 리셋 경계 불일치 리스크 없음
- **확인 필요 — 월간 보스 리셋 경계**: 검은마법사(월간 보스)의 Nexon 서버 리셋 시각(예: 매월 1일 00:00 KST 등)이 아직 실측 확인되지 않았다. 확인 전까지는 [[ADR-004]] 알림 로직에서 월간 리셋 경계 근처의 별도 엣지 케이스 처리를 하지 않는다(주간과 동일한 방식으로만 처리)
- **UI 개편(확정, 2026-07-13, [[ADR-018]])**: 보스 목록은 하나의 카드에 전부 나열하던 방식에서 보스별 개별 카드로 바뀐다. 카드 왼쪽에는 사용자가 직접 제공하는 보스 일러스트(두상이 보이는 큰 그림 — 기존 원형 초상화와 같은 `src/assets/bosses/` 파일을 교체)가 배경처럼 깔리고 오른쪽으로 갈수록 카드 배경에 자연스럽게 녹아든다. 완료 여부는 왼쪽 체크 대신 카드 오른쪽 끝의 `완료` 뱃지로, 난이도는 텍스트 대신 게임 내 난이도 뱃지와 동일한 스타일의 뱃지로 표시한다. 주간/월간 탭의 활성 상태 표시는 컨텐츠 스케줄러와 동일한 pill 스타일을 재사용하고, 드롭다운·탭은 별도 카드로 묶지 않는다. 일러스트별 크기·위치는 보스마다 달라 개별 설정 가능해야 하며, 세부 데이터 구조는 `docs/ARCHITECTURE.md` 참고.
- **파티 관리 (신규, 2026-07-13, [[ADR-019]], 설계만·구현 전)**: 캐릭터+보스+난이도 단위로 파티 인원을 미리 설정해두는 기능. 완료(처치) 여부·주차와 무관한 상시 설정이며, 보스 카드를 탭하면 뜨는 모달에서 입력한다(입력 범위는 기존 `boss-crystal-prices.json`의 `maxPartySize`/`defaultMaxPartySize` 그대로 재사용, 새 게임 데이터 아님). 파티 인원이 2인 이상으로 설정된 보스는 카드에 사람 여러 명 아이콘 + "n인" 배지를 표시하고, 한 번도 설정하지 않은 보스는 솔로(1인)로 취급해 배지를 표시하지 않는다. 화면에는 주간/월간 탭 아래 "전체/솔로/파티" 서브 필터를 추가해, 현재 활성 탭(주간 또는 월간) 안에서만 필터링한다 — 두 축(주간·월간 / 전체·솔로·파티)은 서로 독립적으로 동작. 이 값은 핵심 기능 4(보스 수익 계산기)의 파티원 수 기본값으로도 쓰인다(아래 참고).

### 3. 사냥 타이머
- 30분 카운트다운 타이머(1소재 단위)
- 타이머 진행 중 지정 주기(기본 1분 또는 2분)마다 알림음을 반복 재생해 솔 야누스 재설치 타이밍을 알림
- 5자석펫 등으로 더 긴 주기가 필요한 경우를 위해 사용자가 직접 분 단위 주기를 입력 가능

### 5. 물욕 아이템 드랍 현황
- **화면 구조(확정, 2026-07-09, 와이어프레임 작업 중 발견)**: 이 기능은 단일 화면이 아니라 2단계다 — 드랍 탭 진입 시 먼저 **보스 목록**이 뜨고, 하나를 선택하면 그 보스의 **물욕템 그리드 + 획득 히스토리** 상세 화면으로 들어간다
- **보스 목록 출처 (확정, 2026-07-09, [[ADR-011]])**: 독립 목록이 아니라 보스 스케줄러(핵심 기능 2)와 동일한 Nexon 동기화 캐시에서 가져온다 — 게임 내 스케줄러에 실제로 등록한 주간 보스만 표시(수동 등록 불가, [[ADR-007]] 원칙과 동일). 월간 보스(검은마법사)는 현재 이 목록에서 제외 — 의도된 것인지 확인 필요
- **난이도 무관 아이템 노출 (확정, 2026-07-09, [[ADR-011]])**: 보스 목록·아이템 그리드 어디에도 난이도를 표기하지 않는다. 보스 하나를 선택하면 `item-drop-table.json`에서 그 보스명과 일치하는 모든 난이도(노멀/하드/익스트림/카오스 등) 엔트리의 아이템을 이름 기준으로 합쳐서 보여준다 — 유저가 실제로 처치하는 난이도를 게임 내에 정확히 등록해두지 않았을 수 있어, 난이도로 거르면 실제로 얻은 아이템이 목록에 없을 위험이 있기 때문
- **아이템 이미지 (확정, 2026-07-09, [[ADR-011]])**: 물욕템 버튼은 텍스트가 아니라 이미지로 표시한다. 이미지는 사용자가 직접 준비해 `src/assets/items/`에 추가(파일명 = 아이템명, 공백은 언더스코어로 치환). 아직 이미지가 없는 항목은 플레이스홀더로 표시
- 보스별로 정해진 물욕템 목록을 버튼 형태로 노출 (`src/data/item-drop-table.json`. 사용자가 제공한 보상 정보 전체를 카테고리별(고정 보상/장비/소비 아이템/주문서/기타/최초 격파)로 보유하며, 이 중 실제로 버튼화할 항목은 사용자가 선별 예정)
- 버튼 클릭/터치 시 해당 아이템을 "획득"으로 기록 (획득 일자 포함). 컨테이너형이 아닌 일반 아이템은 확인창 없이 탭 즉시 기록되고 토스트로 피드백한다(제안, `docs/UI_GUIDE.md` 미반영 — 문구·노출 시간 미정)
- 캐릭터별/보스별 드랍 히스토리 조회 가능
- **확정**: 잘못 등록한 획득 기록은 히스토리에서 개별 삭제/취소 가능. 삭제 시 확인 다이얼로그를 넣을지는 미정(즉시 삭제 vs 확인 후 삭제)
- **확정(2026-07-09) — 랜덤 컨테이너 아이템 결과 기록 ([[ADR-010]])**: "OO옥의 보스 반지 상자"·"혼돈의 칠흑 장신구 상자"처럼 개봉 결과가 확률로 정해지는 아이템은 "획득" 기록 시 실제로 나온 결과(레벨·반지 종류, 또는 장신구 종류)를 후보 목록에서 선택해 함께 기록한다(확률로 자동 추정 금지). 후보 목록은 `src/data/boss-ring-boxes.json`(녹옥/홍옥/흑옥/백옥/생명의 5종, 확률표 반영 완료) + `src/data/accessory-boxes.json`(혼돈의 칠흑 장신구 상자·메이린의 칠흑 장신구 상자 — 이름만 다르고 내용물 동일, 후보 7종 반영 완료. 개별 확률은 게임 내 비공개라 목록만)
- 물욕템 획득 기록은 아이템별 고정 시세로 환산해 핵심 기능 4(보스 수익 계산기)의 수익 합계에 포함된다([[ADR-010]])

### 6. 설정 화면 (2026-07-12, 다른 기능 설명에 흩어져 있던 요구사항 정리)
이 화면을 정면으로 다룬 논의는 아직 없었고, 지금까지 온보딩·에러 처리·알림 등 다른 기능을 설명하며 곁다리로 언급된 요구사항들을 처음으로 한곳에 모은 것이다. 화면 레이아웃·진입 경로 자체는 미정 — "확인이 필요한 사항" 참고.
- **API 키 재입력/변경**: 최초 입력은 온보딩에서 하지만([[ADR-007]]), 이후 키를 다시 입력하거나 바꾸는 것은 이 화면에서만 가능하다. API 키가 무효화된 경우(401/403) 다른 화면에 뜨는 배너가 이 화면으로 재입력을 유도한다([[ADR-008]])
- **계정(메이플 ID) 변경**: 온보딩에서 선택한 계정을 이후 언제든 이 화면에서 바꿀 수 있다(확정, 2026-07-09 — 본 문서 "확인이 필요한 사항" #16, [[ADR-007]] 결정 3). 계정을 바꾸면 화면에 노출되는 캐릭터는 새 계정 것으로 전환되지만, 이전 계정 캐릭터 기준으로 쌓인 로컬 기록(보스 수익·드랍 히스토리)은 삭제하지 않고 그대로 보존한다([[ADR-008]] 참조 무결성 원칙)
- **크래시 리포팅 opt-in 토글**: 기본값 OFF, 켠 사용자에 한해서만 Sentry 등으로 익명 스택트레이스를 전송한다. 캐릭터명·API 키 등 개인 식별 정보는 전송하지 않는다(확정, 2026-07-09, [[ADR-008]])
- **알림 권한 재요청 안내**: 최초 요청 시 거부했거나 이후 OS 설정에서 알림 권한을 껐을 경우, 이 화면에서 다시 요청하도록 안내한다(`docs/ARCHITECTURE.md` 에러 핸들링 표)
- **미예약 알림 개수 표시**: 캐릭터 수 × 알림 항목이 64개 한도를 넘어 일부가 예약되지 못한 경우, 그 개수를 이 화면에 표시한다([[ADR-004]])
- **테마 선택 (범위 포함 확정, 2026-07-12, [[ADR-009]] 재개)**: 레테(다크)·렌(라이트) 중 하나를 사용자가 직접 골라 즉시 전환할 수 있다. 테마 이름과 실제 캐릭터 직업(전직)의 매핑은 여전히 미정이라, 직업 기반 자동 적용이 아니라 두 테마 중 수동으로 고르는 최소 범위 UI로 한정한다
- **화면 하단 — 앱 버전·카피라이트·NEXON Open API 출처 표기 (확정, 2026-07-13)**: **정정** — ~~NEXON Open API 출처 표기는 설정이 아니라 앱 footer에~~ → 이 화면 맨 아래에 앱 버전(`package.json`), 카피라이트("© {연도} 메이플 루틴"), 그리고 이용약관 제6조④가 요구하는 영문 문구 "Data based on NEXON Open API"를 그대로 표시한다(약관 원문 요구 문구, 의역 금지). 앱 전역 footer는 만들지 않는다([[ADR-007]] "이용약관 준수 사항" 정정 참고)

## MVP 제외 사항
- 앱 자체 회원가입/계정 시스템 및 클라우드 동기화 — 기록 데이터는 기기 로컬 저장 전용으로 확정 ([[ADR-003]] 참고). Nexon 개인 API 키도 기기에만 저장하며 별도 서버를 두지 않음
- NEXON Open ID(OAuth) 로그인 — "메이플스토리 파트너스" 승인이 필요한 제한된 기능이라 MVP에서 사용하지 않음 ([[ADR-007]], 2026-07-09)
- 별도 공개 웹 서비스(브라우저에서 접근 가능한 랜딩/정보 사이트) — 보류. Capacitor 앱(WebView) 제공에 먼저 집중하고, 추후 필요 시 별도 검토
- 메소/결정 시세 자동 연동 — 가격표는 수동 입력 또는 정적 데이터 기반
- 파티원 매칭, 랭킹 등 소셜/커뮤니티 기능
- 다국어 지원 (한국어 우선)
- 데미지 계산기, 스타포스 계산기 등 스케줄링/기록 이외의 다른 유틸리티

## 에러/엣지 케이스 처리 원칙 ([[ADR-008]])
세부 정책은 ADR-008·ARCHITECTURE.md 참고. 제품 차원의 원칙만 요약:
1. **실패는 숨기지 않는다**: 동기화/저장이 실패하면 조용히 이전 상태를 유지하는 대신, 무엇이 실패했고 마지막으로 성공한 게 언제인지 사용자에게 보여준다.
2. **빈 상태는 에러가 아니다**: 게임에 아직 아무것도 등록 안 한 신규 캐릭터의 "표시할 항목 없음"은 에러 화면이 아니라 다음 행동을 안내하는 빈 상태 화면으로 다룬다.
3. **과거 기록은 지우지 않는다**: 게임 데이터(보스 목록 등)가 바뀌어도 이미 쌓인 사용자의 기록(수익, 드랍)은 보존한다. 다만 실수로 잘못 남긴 기록(물욕템 획득 등)은 사용자가 직접 지울 수 있어야 한다.
4. **자동 재시도는 최소화한다**: 네트워크 실패 시 백그라운드에서 무한 재시도하지 않고, 사용자가 명시적으로 새로고침하도록 한다(배터리 소모 방지). 단, 알림 발송 판단만은 예외로 실시간 재확인한다([[ADR-004]]).
5. **크래시 리포팅은 opt-in (확정, 2026-07-09)**: 서버가 없는 로컬 전용 앱이라 실제 사용자에게 발생하는 문제를 개발자가 알 방법이 없다는 현실적 문제를 해소하기 위해, 설정 화면에 "익명 오류 보고" 토글(기본값 OFF)을 두고 켠 사용자에 한해서만 Sentry 등으로 스택트레이스를 전송한다. 캐릭터명·API 키 등 개인 식별 정보는 전송하지 않는다 ([[ADR-008]]).
